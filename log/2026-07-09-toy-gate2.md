# 2026-07-09 (evening) — Karpathy gate (ii): (context, target) pair overfit

## Hypothesis
The conditional pathway (deep-sets encoder -> velocity head) trains and ROUTES
context information. Design note: with a single pair the model could ignore
tokens entirely and still overfit, so the gate uses TWO pairs trained jointly
— passing both plus a swapped-context degradation control is the minimal
proof that conditioning works.

## Setup
scripts/gate2_context_pair.py: two d=4 GMM targets (seeds 1042, 1142), one
frozen-protocol context each (K=128), ICSModel (default sizes: enc 128/256,
head 256x3), 20k steps batch 512 mixed across pairs, cosine 1e-3. Eval per
pair via ics_evaluate (augmented-space certificate) + sliced-W2 + swapped
control (sample with the other pair's context, de-whitened by that context's
transform, compared to this pair's target).

## Pre-registered PASS criteria
Both pairs: ESS/N(2N) >= 5%, stable, |logZ-hat| <= 0.1, SW2^2 <= 3x same-p
floor; AND swapped-context SW2^2 >= 3x right-context SW2^2 on both.

## Expectation
PASS. Risk: the shared head might average the two targets if the encoder
embedding is uninformative — that shows up as BOTH pairs failing SW2 and the
swap control showing no separation.
