# 2026-07-09 — validation gates (tests-first)

## Hypothesis
The four CLAUDE.md validation gates can be written as closed-form pytest checks
before any estimator code exists, and a numpy implementation can pass all of
them without tuning fudge factors.

## Setup
- Env: reused `~/wl-challenge-env` (python 3.11.5, numpy 2.4.2, scipy 1.17.0,
  matplotlib 3.10.8), pytest 9.1.1 installed into it.
- Tests written first (commit 1db9710), then `stage0/` implemented:
  `utils` (logsumexp), `is_estimators` (ESS/N, logZ, streaming summary,
  Gaussian closed forms), `targets` (Gaussian, GMM, Funnel, DoubleWell — all
  with analytic gradients), `mala` (vectorized chains, tempering),
  `posterior` (grid + adaptive-tempering SMC + energy-observation
  pseudo-likelihood), `sliced_w2`, `stability` (N-doubling).
- Closed forms derived independently in the test files:
  shift ε per dim: ESS/N → exp(−d ε²); proposal variance (1+ε): ESS/N →
  (1+2ε)^{d/2}/(1+ε)^d (valid ε > −1/2). Both are exp(−D₂(p‖q)), D₂ = Rényi-2.

## Expectation
All gates pass at the stated tolerances; MC tolerances chosen from analytic
standard errors so passes are meaningful, seeds fixed so runs are deterministic.

## Result
`29 passed in 3.90s` — gates 1–4 plus auxiliary validators (logsumexp vs scipy,
streaming ≡ direct IS accumulation, finite-difference gradient checks for all
four target families, sampler moment checks, MALA moments + temperature
scaling, sliced-W2 vs ‖δ‖²/d, energy-context posterior peak-at-truth).

**Finding worth keeping (affects M1 design):** the empirical-ESS diagnostic has
its own sampling noise with relative sd √((E w⁴/(E w²)² − 1)/N). For mean shift
this is √((e^{4dε²}−1)/N): at (d=16, ε=0.5), N=3·10⁵ it is ≈ 540%, i.e. the ESS
estimate near/beyond the frontier is order-of-magnitude only and typically an
OVERestimate (the dominating rare weights are missed). For a narrow proposal
(variance scale s² ≤ 3/4) E[w⁴] diverges outright. Consequences:
1. Two originally-planned strict gate-1 cases were moved to reliable-regime
   points; a dedicated `test_deep_tail_regime_...` pins (16, 0.5) to an
   order-of-magnitude band only. This is a test-selection correction made
   BEFORE the gates went green, documented here rather than silently.
2. M1 must (a) use the analytic surface where available, (b) at empirical
   points report the estimated reliability regime, (c) treat "empirical ESS
   looks fine" as untrustworthy exactly in the near-vacuous regime — which is
   the regime the certificate cares about. The certificate story in RESULTS.md
   must carry this caveat: a *low* measured ESS is trustworthy bad news; a
   *moderate* measured ESS deep in the mismatch regime can be false comfort.

## Updated belief
Estimator stack is trustworthy inside its validated regime. Proceed to M1 with
N = 10⁶ per grid point and per-point reliability flags (4th-moment criterion,
analytic where available); mark grid points where even N=10⁶ cannot certify the
ESS estimate.
