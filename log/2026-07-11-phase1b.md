## Readout A — eval_curve (submitted dependent on the chain)

scripts/eval_curve.py evaluates {gate3e-200k, 0.25M, 0.5M, 1M, 1.5M, 2M} with the
two frozen instruments, identical seeds to the toy phase so numbers are directly
comparable:
- TT block = gate3e eval protocol verbatim: 24 trained targets (first 2/cell of the
  128 zoo), ctx seeds fold_in(key(909000+jj), T), eval seeds 800000+100jj+T,
  bespoke refs 700000+100jj (computed once, shared across checkpoints); funnels K=512.
- Fresh-θ block = paired_eval protocol verbatim: 24 fresh targets (key(424242)
  fold-ins), ctx seeds key(515151) fold-ins, refs 600000+1000ci+100i (once, shared),
  eval seeds = the z128 slot (900000+1000ci+100i+T+10) so the 200k column reproduces
  paired_eval.json's z128 numbers.
- Both (K,T) columns; T=1 primary per PLAN. Per-row: certificate (ESS@N/2N, stable,
  logẑ, d2_hat = −ln ESS_frac/d with the stable flag as reliability), SW2²,
  ratio-to-bespoke, mode_recovery, composite pass (ESS≥1% ∧ stable ∧
  SW2 ≤ max(2×ref, 0.1)).
- Missing snapshots are skipped LOUDLY (logged in json + stdout), never silently.

**Expected outcome:** TT T=1 composite fraction rises monotonically-ish with steps
(the H-starve signature); the decisive numbers are TT@2M vs 50%, FQ = fresh median
ratio(200k)/ratio(2M) vs 4×, FE = any d=4 family median certified ESS ≥ 5% (T=1).
Pre-registered branch distribution: **ALIVE 45% / MEMORIZE 25% / STRUCTURAL 30%**;
P-curve (70%): TT curve still visibly rising at 2M (no plateau by the frozen
<10%-over-final-500k rule).

## Readout B — cost-crossover (scripts/readout_b.py at 200k and 2M)

12-target subset + contexts verbatim from $SCRATCH/ics-zoo/eval/eval_set.pkl
(context_t5, as baselines.py used). Frozen metric definitions; the accounting
procedure (free periphery, documented here BEFORE the run):
- ICS all-in per new target = wall-clock of forward sampling (2N=8192, n_ode=100)
  + weights (CNF log-density backward pass) + certificate incl. doubling check.
  Measured per target, jit warm-up EXCLUDED via a discarded first target
  (compile time reported separately, same convention as B4's adapt/compile split).
- Crossover on target j: SW2²_ICS(j) ≤ SW2²_B4(j) AND allin_ICS(j) <
  adapt_seconds(j) + sample_seconds(j) from results/baselines.json (frozen
  references, not rerun).
- Certified eff. samples/s (ICS) = ESS_frac_2n × 8192 / allin. MCLMC comparator =
  8192 / (adapt+sample) — an UPPER bound on its effective rate (credits the scored
  8192 draws as fully effective; generous to MCLMC, one-sided-safe, consistent with
  the B2/B4 precedent).
- Run at BOTH gate3e-200k and 2M so the within-line (128-zoo) comparison is clean;
  the frozen baseline value 3/12 (train4 line) is reported alongside, labeled.

**Expected outcome:** P-crossover (65%): 3/12 → ≥5/12 at 2M. Decision-rule use:
crossover ≥ 6/12 modifies pivot options only.

## Readout C — usefulness barometer (amendment; eval-only, NON-GATING)

Three real targets, zero-shot, standard context protocol (4 MALA chains, K=128,
(x,E,∇E) via autodiff, T ∈ {1,5} columns):
(a) **eight-schools non-centered** d=10: (μ, log τ, z₁..₈); μ~N(0,5²),
    τ~HalfCauchy(5) via log τ + Jacobian, y_j~N(μ+τz_j, σ_j²), classic data.
(b) **inference-gym Banana** d=2 (curvature 0.03, Haario form), exact hand-port —
    the gym package is TFP-bound and incompatible with our jax 0.9.1 env; the target
    density is closed-form. CAVEAT logged: geometry-adjacent to our held-out banana
    zoo family (still never trained on). Real-data gym standards at d≤10 do not
    exist; NealsFunnel would duplicate (a)'s geometry — choice documented here.
(c) **WL band-power posterior** d=3 (Ω_m, σ₈, n_s): Euclid-like 120 band powers +
    Knox covariance from eft-sbi phase-0.5 (read-only; CAMB grid job runs with that
    repo's .venv, cache redirected to OUR scratch). Differentiable surrogate:
    per-band polynomial fit over the box Ω_m 0.27±0.12, σ₈ 0.83±0.15, n_s
    0.94±0.08 on a 9³ CAMB grid (box center ≠ fiducial by construction, so the
    prior mode does not sit on the answer; fid lands at ≈ (+0.25, −0.20, +0.28)
    halfwidths — CORRECTED pre-launch from an earlier fid-centered draft of this
    entry, before any WL job was submitted), validated on held-out grid points
    (report max
    rel. error; require < 1% of the band's Knox σ to proceed — else refit higher
    order). Sampling space: u ∈ R³ with θ = center + halfwidth·tanh(u/2.5)
    (practitioner-adapter transform; Jacobian included; NO extrapolation possible).
    Prior: u ~ N(0, I). Data vector: C(θ_fid) + Knox noise draw, seed
    zlib.crc32(b"wl-bandpower-data") — fid deliberately NOT at box center offsets
    the posterior from the prior mode. Reference and ICS sample the SAME surrogate
    posterior; surrogate-vs-CAMB fidelity reported separately.
- References: blackjax NUTS + window_adaptation, 4 chains × 20k draws (after 2k
  adaptation), split-R̂ < 1.01 on every coordinate required. The R̂ estimator and
  the NUTS pipeline are validated on a closed-form 3-d correlated Gaussian FIRST
  (moment match < 2% and R̂ < 1.01 gate the script; CLAUDE.md validation rule).
- Metrics per target: certified ESS@N/2N + stability flag, SW2² vs reference draws
  (u-space = sampling space; θ-space moments also reported for (c)), Readout-B-style
  all-in cost accounting, NUTS reference cost for context.
- Evaluated at gate3e-200k and 2M.

**Pre-registered (60%):** at 2M, ≥1 real target (most likely the d=3 band-power)
yields stable certified ESS ≥ 1% zero-shot. NON-GATING for all branches.

## Post-mortems, first wave (all before any Readout-C number exists)

1. **rg31703 silently ate two GPU jobs** (15739210 scalingeye, 15739211
   rb200k): both co-scheduled on that node seconds after the maintenance
   window, both emitted only the XLA plugin's ptxas banner and then nothing
   until TIME LIMIT — consistent with a wedged GPU hanging the first CUDA call
   (the identical eval ops ran at ~3 s/target in paired_eval jobs). The 2M
   chain on rg31702 streams normally. Resubmitted with --exclude=rg31703 and a
   start-up heartbeat in readout_b for diagnosability. If they hang on a
   DIFFERENT node, the node theory is wrong and the code path gets audited.
2. **wl_surrogate GATE-FAIL, diagnosed as grid-side noise**: held-out error
   plateaued at ~0.3 Knox σ for degrees 5–8 (truncation error would fall with
   degree; a plateau means point noise in the CAMB grid itself). One retry
   pre-registered-consistent ("refit higher order" exhausted → attack the
   noise): engine resolution doubled (nz_int 1200, nz_pk 80, nk 800, n_ell
   480) in OUR wl_grid.py (ClEngine constructor args; eft-sbi untouched).
   If the retry still fails the 1% gate, the gate gets amended with full
   documentation — note the internal ICS-vs-reference comparison is unaffected
   by surrogate fidelity by construction (both sample the same surrogate).
3. **Banana reference under-converged at 4×20k NUTS draws — the validation
   gate caught it** (that is what it is for): variance vs the exact sampler
   swung −15% (20k) to +13% (200k) with R̂ ≈ 1.002 throughout — huge IACT
   along the arch; R̂ alone is blind to it. Fixes: (a) the banana SW2
   reference is now its EXACT transform sampler (strictly better; NUTS kept
   as a 4×1M machinery cross-check with MC-error-calibrated gates 0.03 mean /
   0.10 var); (b) all references now also pass the repo-standard half-vs-full
   variance stability check (< 3%); (c) draws raised: eight-schools 4×100k,
   WL 4×50k. rc_refs.sh got `set -e` (the surrogate failure had fallen
   through to the refs step — masked only because the missing npz was
   skipped loudly).
4. wl_grid #1 (15739448) COMPLETED in 786 s — the retry reuses nothing from
   its cache (resolution is in the cache key).
